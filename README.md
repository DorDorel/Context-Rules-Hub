## âš¡ï¸ Quick Setup (The Alias)

To use this tool from anywhere in your terminal, set up a permanent alias.

1. **Clone this repository** to a permanent location (e.g., `~/dev/ai-rules`).
2. **Edit your shell configuration** (`.zshrc` or `.bashrc`):
   ```bash
   nano ~/.zshrc
   ```
3. **Add the following line** (update the path to where you cloned this repo):
   ```bash
   alias rules-sync='python3 ~/dev/ai-rules/sync_agents.py'
   ```
4. **Refresh your terminal**:
   ```bash
   source ~/.zshrc
   ```

## ğŸš€ Usage

Navigate to any project folder on your machine and run the sync command with your desired stack.

### Syntax

```bash
rules-sync <stack_name>
```

### Examples

**For a SwiftUI Project:**

```bash
cd ~/my-apps/awesome-ios-app
rules-sync swiftui
```

**For a Flutter Project:**

```bash
cd ~/my-apps/cross-platform-app
rules-sync flutter
```

_(Note: This will combine both `core.md` and `rules.strict.md` from the flutter folder)_

**For a NestJS Project:**

```bash
rules-sync nest
```

## â• Adding New Stacks

To add a new technology stack (e.g., Python):

1. **Create a new folder**: `python_rules`.
2. **Add a `core.md` file** inside with your instructions.
3. **(Optional)** If the folder name doesn't match the command exactly, update the `TECH_MAP` in `sync_agents.py`.

> **Note:** All generated files include a signature: `> Auto-generated by sync_agents.py.`
